Auditor Document Requests

How this tab works

What this tab is for. The Planning and Execution tabs cover everything StudyTrust prepared in advance. But once TPCSA is in the room, they will almost always need something that was not on the original checklist. This tab is where those additional requests land and are tracked from start to resolution. Every request the auditor submits arrives here immediately.

How it connects to what came before. If a document was partially covered in Planning or Execution but the auditor needs a different version or a specific period, resolve it here and note the variation in the Notes field. There is no need to duplicate the entry across both tabs.

The workflow, step by step.

  1. Open. A new request arrives from the auditor. The status is Open. This is your notification that something is needed.
  2. Assign it. Click the Assigned To field and type the name of the person who can source or prepare that document. Change the status to In Progress. This tells the auditors the request has been received and someone is on it.
  3. The auditor confirms. Before the assigned person starts working, the auditor should set their request to Submitted so it is clear the request is complete and final. If they do not, you can move it to In Progress directly.
  4. Prepare and upload the document. Work with the assigned person to gather the document. Use the Upload button on any row to send the file directly to the Audit {year}/Auditor Requests/ folder in SharePoint. The link fills in automatically.
  5. Resolve it. Set the status to Resolved and enter today's date in Date Resolved. The auditor receives a Teams notification with the link. They do not need to keep checking back.
  6. Close it. Once the auditor confirms the document meets their need, the request is set to Closed. Closed requests remain visible for the audit record but are visually distinct.

How it connects to what comes next. After the audit, review any requests that were raised repeatedly or point to a gap in the standard checklist. These belong in the Recommendations tab so they become Planning items for next year and the same gap is not found again.

Audit Questions

What this tab is for. Use this tab for information that auditors request verbally or in writing, where the answer is text only — no document to upload. This includes anticipated questions based on prior years, questions already answered in previous audits, and clarifications the audit team asks during fieldwork. Each row has a Question, a Source (Anticipated / Prior Year / Auditor), an Answer, a Status, and a Responsible person. Nothing here requires a SharePoint link.

Personnel Documents

🔒 Restricted access. This tab is visible only to members of Dashboards - HR-Authorised-Personnel (Dirk Lourens, Murray Hofmeyr, Phila Mkandla, Engelinah Mtande, Siphokazi Kwakweni). Rows marked Internal Only are not visible to auditors even when they have dashboard access. Change a row to Grant Auditor Access only for the duration of a specific audit request, then change it back to Internal Only once the request is Closed. Simon Letsela manages the SharePoint folder-level permissions separately.
# Description Compliance Date Uploaded Status Responsible Document Category Upload / Link Auditor Access
Auditor Access — How it works. When TPCSA requests a personnel document, change the relevant row to Grant Auditor Access. This does two things: it flags the row as visible in the auditor-facing document list, and it reminds Simon Letsela to grant temporary SharePoint folder access for that specific auditor to the Personnel Contracts subfolder. Once the auditor confirms receipt and the Auditor Request is Closed, change the row back to Internal Only and notify Simon to revoke the folder access. Auditors must never have standing access to personnel documents between audit cycles.

Teams Webhook — Auditor Notifications

When an Auditor Request is set to Resolved, a notification card is posted to this Teams channel. The auditor does not need to keep checking the dashboard.
How to get the webhook URL: In Teams, open the auditor channel → ••• → Workflows → Post to a channel when a webhook request is received → copy the URL.
